# i18n — English + Dutch with a language toggle

Post-Ch12 feature. The app shipped Dutch-only via one catalog; this adds English for
**all interfaces, the public portal, and the generated PDFs**, plus a persistent top-left
language toggle that defaults to Dutch.

## How it works

- **Locale = cookie.** `lgj_locale` (`nl`|`en`, absent → `nl`). A cookie (not localStorage) so the
  same choice drives server components, the public portal, the PDF route handlers, and `proxy.ts`.
- **Catalogs.** `messages/nl.ts` is the source of truth. `messages/types.ts` widens its `as const`
  literals to a `Messages` type, so `messages/en.ts` (`export const en: Messages`) is
  **completeness-checked** against every Dutch key at compile time. `messages/index.ts` exposes
  `messages: Record<Locale, Messages>`, `DEFAULT_LOCALE`, `isLocale`.
- **Access.** Server: `i18n/server.ts` `getLocale()` (cookie via `next/headers`, `cache()`-wrapped) +
  `getMessages()`. Client: `i18n/client.tsx` `LocaleProvider` (mounted in the root layout with the
  server-resolved locale) + `useMessages()`/`useLocale()`. Dates: `i18n/format.ts`
  `formatDate`/`formatDateTime` (`nl`→`nl-NL`, `en`→`en-GB`).
- **Switch.** `i18n/actions.ts` `setLocale` server action sets the 1-year cookie; `LanguageToggle`
  (top-left of the sidebar, mobile bar, portal header, and login) calls it then `router.refresh()`.

## Migration pattern

Each file dropped `import { nl } from "@/messages/nl"` and rebound `nl` locally — **bodies
unchanged** (`nl.x.y` references kept):
- Client components: `const nl = useMessages();`.
- Server components/pages/display: `const nl = await getMessages();` (made `async`; cookie read makes
  them dynamic — all were already `ƒ`).
- Date sites: `const locale = …` + `formatDate(x, locale)`.
- **PDFs** (`lib/pdf/*`) are rendered outside the RSC tree → the route handler resolves `m`/`locale`
  and passes them as **props** to the document components + `DocHeader`/`Footer`; `feedbackSections`
  takes `m`.
- `proxy.ts` reads the cookie off `request.cookies` directly.
- `global-error.tsx` has no provider (it replaces the root layout) → reads the cookie client-side
  after mount (default locale on first paint to avoid a hydration mismatch).

## Scope

In: every visible UI label, the portal, the three PDFs, plus the previously-hardcoded UI strings
(dashboard, login, nav, "year", share-prompt) moved into the catalog. **Out (still Dutch):** zod
validation messages + server-action error fallbacks (`features/**/schema.ts`, `**/actions.ts`) — a
deeper layer, left for a follow-up. Kata names / proper nouns are never translated. Name-sort
`localeCompare(…, "nl")` left as-is (server query, negligible).

## Build

Fanned out via a Workflow: 8 parallel section translators → 1 assembler wrote `en.ts` → 7 parallel
migration groups (43 files). The PDF group, `proxy.ts`, `global-error.tsx`, and `loading-screen.tsx`
were migrated by hand (prop-threading / no-provider specials). The `Messages` completeness type + the
full gate suite caught the few agent misses.
